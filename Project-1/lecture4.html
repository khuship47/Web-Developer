Layered Architecture

  • A protocol is a set of rules that both side of the networks use in order to communicate.
  • These protocols are implemented in every OS which makes the web development faster.

Link Layer

  • It is the lowest layer, responsible for both the physical transmission of data across media and establishing logical links.
  • It handles issues like packet creation, reception, error detection, collisions, etc.
  • Media access control (MAC) referred to as a hardware address, is a unique, 12-char alphanumeric attribute that is used to identify individual electronic devices on a network.

Internet Layer

  • It routespackets between communication partners across networks. It is also called IP layer.
  • It sends out message to the destination, but does not guarentee that message will reach without loss of information.
  • The internet uses Internet Protocols (IP) addresses, which are numeric codes that uniquely identifydestinations on the internet.
  • Every device using internet has an IP address.
    • In (IPv4), four 8-bit integers separated by . encode the address.
    • In (IPv6), eight 16-bit integers and has over billion times the number in IPv4.
    • Port Address Translation (PAT), allows multiple, unrelated networks to make use of the same IP address.

Transport Layer

  • It ensures transmissions arrive in order and without any error.
  • The data is first broken into small packets accoridng to Transmission Control Protocol (TCP).
    • Each data packet includes a header with a sequence number.
    • The receivers will put the header number in sequence to retrieve the original message.
    • Each packet sends acknowledgement once the message arrives back successfully.
    • If no acknowledgement is received, then the packet will be retransmitted.
  • User Datagram Protocol (UDP) is used in lieu of TCP, when little lose of data does not really affect the message. An example could be live broadcast of a football game.

Application Layer

  • It implements process-to-process communication.
  • It includes the following protocols:
    • HTTP: Hypertext Transfer Protocol is used for web communication.
    • SSH: Secure Shell Protocol allows remote command-line connections to servers.
    • FTP: File Transfer Protocol is used for transferring files between computers.
    • POP: Email-related protocols for transferring and storing email.
    • DNS: The Domain Name System protocol used for resolving domain names to IP addresses.

Domain Name System

DNS Overview

  • A computer containing a database of public IP addresses associated with the name of the website. This is called Domain Name System.
  • By separating the domain name of a server from its IP address, a site can move to a different host without changing its name.
  • All domain names have at least a top-level domain(TLD) name and a second-level domain (SLD) name.
  • The rightmost portion of the domain name is the top-level domain.
  • Generic top-level domain:
    • Unresticted: TLDs include .com, .net, .org, and .info.
    • Sponsored: TLDs include .gov, .mil, .edu, etc.
    • New: contains .art, .jobs, .tax, and many more.
  • Country code top-level domain:
    • It comes under the control of the countires which they represent, so they are administered differently.
    • In the United Kingdom, businesses have co.uk, while anyone doing business in Canada can use .ca domain.
    • Some countries have allowed unrestricted use of domain to generate revenue.
    • International top-level domain name allows domains to use non-ascii characters.
  • Name registration process
    • Domain name registrars is a special organization which manages the registration process.
    • Registrant searches on web portal for the domain.
    • Registrant queries the relevant TLD Registry Operator to see if the requested domain is available.
    • If domain in available, then pay for the domaina dn provide all the necessary information.
    • Registry operator adds all the WHOIS information for the new domain to its authoritative list.
    • Registry operator will push DNS 5 information for new domain out to its name servers for the TLD.

Uniform Resource Locator

Information about URL

  • Uniform resource locators (URLs) allow clients to request particular resources from the server.
  • Required components
    • The protocl used to connect, usually http/ https.
    • The domain to connect to i.e., www.domain_name.com.
  • Optional componets
    • Port: It is a software connection point used by the underlying TCP/IP protocol and the connecting computer. Port 80 is the default port for web-related HTTP requests. It is added after the domain name with ':'.
    • Path: It identifies a file or directory to access on that server. When requesting a folder, if the path is not mentioned, the web server will decide which file to send you.
    • Query String: It is a critical way of passing information, such as user form input, from the client to the server. They are encoded as key value pairs delimited by '&' and preceded by '?' symbol.
    • Fragment: This is used as a way of requesting a portion of a page. Browsers will see the fragment in the URL (denoted by #), seek out the fragment tag anchor in the HTML, and scroll the website down to it.
