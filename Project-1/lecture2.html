Definitions and History of Web

The Birth of the Web

  • Sr. Tim Berners-Lee published main features of web in 19921
  • A Uniform Resource Locator (URL) to uniquely identify a resource on the WWW.
  • The Hypertext Transfer Protocol (HTTP) to describe how requests and response operate.
  • A software program (web server software) that can respond to HTTP requests.
  • Hypertext Markup Language (HTML) to publish documents.
  • A program (browser) that can make HTTP requests to URLs and that can display the HTML it receives.

Web Applications vs Desktop Applications

Advantages

  • Can be accessed from any Internet-enabled computer.
  • Can be used with different OS and browser applications.
  • Easier to update as only the server needs to be updated, and not every computer in the buidling using that software.
  • Centralized storage on the server, means less security problems than storing locally.

Dissdvantages

  • Required to have an active Internet connection.
  • Security concers about sensitive data being transmitted over internet.
  • Problems with storage, licensing, and use of uploaded data.
  • Certain websites might not have the same appearance across all the browsers.

Types of Networks

Circuit switching: In the past, phones were routed through operators who physically connected the caller and the receiver by connecting a wire to a switchboard to complete the circuit. It was difficult to scale and bandwidth was used inefficiently.

Packet switching: It coes not require a continuous connection. The message was sent in packets with a number at the start, so when receiver gets the message the numbers are put together to complete the message.

Addition Information

  • Web Development can be understood as a complex ecosystem that builds on existing technologies and contributes new protocols and standards that facilitates client-server interaction.
  • Web pages evolved from static to dynamic, means now web pages are hosted on a server which lets websites generate content dynamically.
  • Programs are needed because to make things flexible, so when need to revise the design it becomes easier.
  • Web development today is thus more complicated than it was when initially started.

Client-Server Model

Definition

Client machines: These are the desktops, laptops, smart phones, and tablets or anythign you can access a web page from.

  • This includes OS, processing speed, screen size, available memory, and storage

Server machines: These are the machines that hosts web applications, stores user and program data, and performs security authorizathion tasks.

  • Powerful machines to handle high amount of traffic and bandwidth at a time.
  • Main use of server is to listen for requests, process it, and responds with an appropriate message.

Types of Servers

  • Web servers: A computer system capable of delivery web content to end users upon requests via web browser.
  • Application servers: It acts as intermediate between OS and main server.
  • Database servers: These are dedicated for data retrieval from the database.
  • Mail servers: A program that sends and recieves email.
  • Media servers: It is used for applications tha stores and share multimedia files.
  • Authentication servers: It checks that the entity tried to access the network is allowed or not.
  • Cloud servers: It offers elastic provisioning of virtual servers, which scales costs and hardware to the demand.
